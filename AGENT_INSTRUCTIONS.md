# Инструкция для агента

Документ описывает, как корректно решать и оформлять задания по курсу Кушнера в этом репозитории. Если вы агент (Claude, ChatGPT, Cursor, любой другой), прочитайте этот файл полностью перед тем как что либо делать.

## Контекст курса

Курс посвящен геометрии дифференциальных уравнений. Темы:

1. Управляемость нелинейных систем (теорема Рашевского Чоу, коммутаторы векторных полей, потоки).
2. Конечномерные динамики эволюционных уравнений первого порядка (инвариантные ОДУ в джет пространстве, контактная форма, условие совместимости через производную Ли).

Все задания решаются в Maple с пакетами `DifferentialGeometry`, `JetCalculus`, `LinearAlgebra`, `PDETools`, `Tools`.

## Жесткие правила оформления

Сначала прочитайте `rules-of-work.txt`. Кратко самое важное:

1. Никаких эмодзи, тире, декоративных разделителей, жирного, курсива.
2. Краткий ответ в 1 2 предложения. До 5 коротких пунктов с нумерацией. Без вложенных списков.
3. В коде минимум комментариев. Стандартная библиотека Maple.
4. Никакого лишнего логирования или украшательств.

## Формат сдачи

Каждое задание оформляется тремя файлами в отдельной папке (`hw1/`, `hw2/`, `final/`, ...):

1. `homework_maple_sorokoletov_442.mw` рабочий лист Maple.
2. `homework_maple_sorokoletov_442.txt` тот же код в plain text (без XML), без лишних `print`.
3. `homework_explanation_442.txt` построчное пояснение метода и кода на русском.

Имена файлов не менять. Имя фамилия и номер группы в имени файла обязательны.

## Метод решения

### Тип 1. Управляемость системы двух векторных полей в R^n

1. Записать векторные поля X, Y через `evalDG`.
2. Вычислить коммутатор `Z := LieBracket(X, Y)`.
3. Если задача на поверхности (сфера, цилиндр и т. д.), проверить касательность всех полей к поверхности через `LieDerivative` функции уровня. Это должно быть тождественно 0.
4. Составить матрицу из компонент полей и вычислить ранг распределения. Для двумерной поверхности достаточно показать, что хотя бы одно векторное произведение пар полей ненулевое в каждой точке.
5. По теореме Рашевского Чоу заключить полную управляемость.
6. Построить явный маршрут из точки A в точку B как композицию потоков `Flow(X, t1), Flow(Y, s1), Flow(Z, q1)`. Углы выбирать так, чтобы последовательно занулять/выставлять координаты.
7. Привести 2 3 числовых теста.

Эталоны: `hw1/`, `hw2/`.

### Тип 2. Конечномерные динамики эволюционного уравнения первого порядка

Пусть уравнение имеет вид $u_t = \varphi(x, u, u_x)$.

1. Записать уравнение `E` и `phi` через джет нотацию ($y_0 = u$, $y_1 = u_x$).
2. Задать кандидата на инвариантное ОДУ первого порядка $u_x = f(x, u)$:
   ```
   omega := evalDG(dy[0] - f(x, y[0])*dx);
   ```
3. Записать эволюционное поле `S := evalDG(phi*D_y[0])` и ограничить его на $\{y_1 = f\}$.
4. Вычислить условие совместимости `LieDerivative(S1, omega) &w omega`. Из его раскрытия получить PDE на $f$.
5. Решить PDE через `pdsolve` (метод характеристик).
6. Выбрать простейшую нетривиальную $f$, проверить решением `dsolve` ОДУ $u_x = f$, получить $x$ зависимость общего решения.
7. Через `Flow(S2, t)` и `Pullback` получить $t$ зависимость. Записать кандидата `sub_u`.
8. Проверить: `simplify(eval(E, sub_u))` должно дать 0.
9. Построить графики `plot` и `plot3d` при конкретных значениях параметров.

Эталоны: `examples/Задача 2 Оруджева.mw` (принят), `final/`.

## Шаблоны

В `templates/` лежат:

- `template_finite_dynamics.mw` готовая структура из 25 ячеек Maple под задачи типа 2.
- `template_finite_dynamics.txt` тот же код в plain text.
- `template_controllability.txt` код в plain text под задачи типа 1.
- `template_explanation.txt` каркас текста пояснения.

Стандартный рабочий процесс: скопировать шаблон, заменить только содержательные строки (уравнение `E`, `phi`, ансатц `f`, итоговая подстановка `sub_u`). Структуру и имена переменных не менять.

## Запреты

1. Не выдавать просто общее решение PDE как ответ. Преподаватель явно отвергает такие сдачи. Нужна именно редукция на инвариантное конечномерное подмногообразие.
2. Не упрощать метод до подстановки ansatz без вывода условия совместимости.
3. Не использовать сторонние пакеты помимо перечисленных.
4. Не добавлять разделы, которых нет в эталонных работах (введение, литература, выводы лишние).
5. Не менять структуру именования файлов.

## Проверка перед сдачей

1. `simplify(eval(E, sub_u))` равно 0.
2. Все три файла на месте и совпадают по содержанию (mw и txt идентичны по коду, explanation описывает каждый шаг).
3. Имя файла соответствует шаблону `homework_maple_sorokoletov_442`.
4. Текст пояснения проходит правила из `rules-of-work.txt`.

## Если данных не хватает

Задать максимум 2 уточняющих вопроса и остановиться. Не достраивать постановку самостоятельно.
