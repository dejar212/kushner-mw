Пояснение к коду (Домашнее задание 2: Управляемость на сфере S²)

restart;
- Очищает память программы Maple от переменных предыдущих сессий.

with(DifferentialGeometry):
- Подключает библиотеку дифференциальной геометрии (для работы с векторными полями, скобками Ли и потоками).

with(LinearAlgebra):
- Подключает библиотеку линейной алгебры (для работы с матрицами, определителями, векторными произведениями).

DGsetup([x, y, z], M);
- Задает трехмерное пространство (многообразие) M с координатами (x, y, z). Единичная сфера S² задается уравнением x² + y² + z² = 1 внутри этого пространства.

X := evalDG(-y*D_z + z*D_y);
- Определяет первое управляющее векторное поле X — генератор вращения вокруг оси x. Это поле описывает действие первого двигателя: вращение объекта в плоскости (y, z). Поток Φ_t^X переводит точку (x, y, z) в (x, y·cos(t) + z·sin(t), −y·sin(t) + z·cos(t)).

Y := evalDG(-x*D_z + z*D_x);
- Определяет второе управляющее поле Y — генератор вращения вокруг оси y. Описывает действие второго двигателя: вращение в плоскости (x, z). Поток Φ_s^Y переводит (x, y, z) в (x·cos(s) + z·sin(s), y, −x·sin(s) + z·cos(s)).

Z := LieBracket(X, Y);
- Вычисляет коммутатор (скобку Ли) [X, Y] = X∘Y − Y∘X. Результат: Z = −y·D_x + x·D_y, что является генератором вращения вокруг оси z. Это означает, что чередуя малые повороты вокруг осей x и y, можно получить эффект поворота вокруг оси z. Поток Φ_q^Z переводит (x, y, z) в (x·cos(q) − y·sin(q), x·sin(q) + y·cos(q), z).

sphere := x^2 + y^2 + z^2;
LieDerivative(X, sphere);
LieDerivative(Y, sphere);
LieDerivative(Z, sphere);
- Проверяем, что все три векторных поля X, Y, Z касаются сферы S². Для этого вычисляем производную Ли функции r² = x² + y² + z² вдоль каждого поля. Результат L_V(r²) = 0 означает, что поле V в каждой точке касается поверхности уровня r² = const, т.е. касается сферы. Все три производные равны нулю.

C_X, C_Y, C_Z — векторы компонентов полей X, Y, Z в базисе (∂/∂x, ∂/∂y, ∂/∂z):
- C_X = (0, z, −y), C_Y = (z, 0, −x), C_Z = (−y, x, 0).

Mat := Matrix([[0, z, -y], [z, 0, -x], [-y, x, 0]]);
Det3 := Determinant(Mat);
- Составляем матрицу 3×3 из компонентов полей X, Y, Z (строки — координаты полей) и вычисляем её определитель. Он тождественно равен 0. Это ожидаемо: три вектора, касательных к двумерной поверхности S², не могут быть линейно независимы в трехмерном объемлющем R³.

CrossProduct, DotProduct:
- Для доказательства, что ранг системы {X, Y, Z} равен 2 (= dim S²) в КАЖДОЙ точке сферы, вычисляем квадраты норм векторных произведений всех пар:
  |X × Y|² = z²(x² + y² + z²)
  |X × Z|² = y²(x² + y² + z²)
  |Y × Z|² = x²(x² + y² + z²)
  Сумма = (x² + y² + z²)²
  На сфере S² (где x² + y² + z² = 1) эта сумма = 1 > 0.
  Поскольку сумма строго положительна, хотя бы одно из произведений ненулевое, т.е. хотя бы одна пара полей линейно независима в любой точке S². Значит ранг распределения = 2 = dim S² всюду.

Вывод (теорема Рашевского–Чоу):
- Управляющие поля X и Y вместе со своим коммутатором [X,Y] = Z порождают в каждой точке S² всё касательное пространство. По теореме Рашевского–Чоу это означает полную управляемость: из любой точки на сфере можно добраться до любой другой, используя только два управления (вращения вокруг осей x и y).

F_X := Flow(X, t_param);
- Создаёт поток (однопараметрическую группу) векторного поля X. Это преобразование Φ_t^X — поворот вокруг оси x на угол t.

F_Y := Flow(Y, s_param);
- Создаёт поток поля Y: Φ_s^Y — поворот вокруг оси y на угол s.

F_Z := Flow(Z, q_param);
- Создаёт поток поля Z = [X,Y]: Φ_q^Z — поворот вокруг оси z на угол q. Это ключевое отличие: мы используем поток вдоль КОММУТАТОРА, т.е. направления, которое явно не задано двигателями, но возникает из их чередования. Именно использование потока коммутатора составляет суть задачи (аналогично задаче с тележкой).

Построение управления — композиция трёх потоков Φ_q^Z ∘ Φ_s^Y ∘ Φ_t^X:
- Алгоритм перехода из точки A = (x₀, y₀, z₀) в точку B = (x₁, y₁, z₁) на S² с использованием потоков вдоль всех трёх векторных полей X, Y и Z = [X,Y].

t1 := arctan(-y0, z0);
- Шаг 1 (поток вдоль X): выбираем угол t так, чтобы после поворота вокруг оси x координата y стала равна нулю. Условие: y₀·cos(t) + z₀·sin(t) = 0, откуда t = arctan(−y₀, z₀). После этого шага точка оказывается в плоскости xz: P₁ = (x₀, 0, √(y₀² + z₀²)).

rho1 := sqrt(y0^2 + z0^2);
- Вычисляем ρ₁ = √(y₀² + z₀²) — расстояние исходной точки от оси x в плоскости yz. Это z-координата точки P₁ после первого шага.

rho2 := sqrt(x1^2 + y1^2);
- Вычисляем ρ₂ = √(x₁² + y₁²) — расстояние целевой точки от оси z в плоскости xy. Это будет x-координата промежуточной точки P₂ после второго шага.

s1 := arctan(rho2, z1) - arctan(x0, rho1);
- Шаг 2 (поток вдоль Y): выбираем угол s так, чтобы после поворота вокруг оси y z-координата стала равна z₁. Применяя поток Y к точке P₁ = (x₀, 0, ρ₁), получаем уравнение −x₀·sin(s) + ρ₁·cos(s) = z₁. Его решение: s = arctan(ρ₂, z₁) − arctan(x₀, ρ₁). После этого шага: P₂ = (ρ₂, 0, z₁) = (√(x₁² + y₁²), 0, z₁).

q1 := arctan(y1, x1);
- Шаг 3 (поток вдоль Z = [X,Y], КОММУТАТОР): выбираем угол q так, чтобы после поворота вокруг оси z точка P₂ перешла в целевую точку B. Поток Z вращает в плоскости xy, сохраняя z-координату. Из условий ρ₂·cos(q) = x₁ и ρ₂·sin(q) = y₁ получаем q = arctan(y₁, x₁). После этого шага: FinalPoint = (x₁, y₁, z₁) = B.

P1_expr := ApplyTransformation(F_X, A);
P1 := simplify(eval(P1_expr, t_param = t1)) assuming real;
- Применяем поток X с вычисленным углом t₁ к начальной точке A. Получаем промежуточную точку P₁ = (x₀, 0, ρ₁) — точка лежит в плоскости xz.

P2_expr := ApplyTransformation(F_Y, P1);
P2 := simplify(eval(P2_expr, s_param = s1)) assuming real;
- Применяем поток Y с углом s₁ к точке P₁. Получаем точку P₂ = (ρ₂, 0, z₁) — z-координата уже совпадает с целевой.

FP_expr := ApplyTransformation(F_Z, P2);
FinalPoint := simplify(eval(FP_expr, q_param = q1)) assuming real;
- Применяем поток Z = [X,Y] (коммутатор!) с углом q₁ к точке P₂. Вращение в плоскости xy при сохранении z = z₁ даёт финальную точку (x₁, y₁, z₁) = B.

Числовые проверки:

Тест 1: A = [0, 0, 1] (северный полюс) → B = [1/√2, 1/√2, 0] (точка на экваторе).
  Углы: t = 0 (y уже 0), s = π/2 (повернуть z от 1 до 0), q = π/4 (повернуть в плоскости xy).
  Цепочка: (0,0,1) →[X,0] (0,0,1) →[Y,π/2] (1,0,0) →[Z,π/4] (1/√2, 1/√2, 0) ✓

Тест 2: A = [0, 0, 1] → B = [1, 0, 0].
  Углы: t = 0, s = π/2, q = 0 (вращение в xy не требуется).
  Цепочка: (0,0,1) →[X,0] (0,0,1) →[Y,π/2] (1,0,0) →[Z,0] (1,0,0) ✓

Тест 3: A = [0, 1, 0] → B = [1/√3, 1/√3, 1/√3].
  Углы: t = −π/2 (убрать y₀=1 из y-координаты), s = arccos(1/√3), q = π/4.
  Цепочка: (0,1,0) →[X,−π/2] (0,0,1) →[Y,arccos(1/√3)] (√(2/3),0,1/√3) →[Z,π/4] (1/√3,1/√3,1/√3) ✓

В трёх тестах финальная точка совпадает с целевой, что подтверждает корректность построения маршрута как композиции потоков вдоль X, Y и Z = [X,Y].
